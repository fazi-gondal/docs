The `/api/tiktok/info` endpoint returns the raw data object from the TikWM API for a given TikTok URL. Unlike [`POST /api/metadata`](/api-reference/metadata), which returns a normalized summary, this endpoint returns the complete TikWM payload — including multiple playback URLs at different quality levels, music information, author profile data, and engagement counts. Use this when you need fine-grained control over which video quality to download or when you want to display rich metadata in your app.

**Endpoint:** `GET /api/tiktok/info`

## Request parameters

<ParamField query="url" type="string" required>
  The full TikTok video URL. Accepts `tiktok.com`, `vm.tiktok.com`, and `vt.tiktok.com` links. Returns HTTP 400 if the URL does not belong to TikTok.
</ParamField>

## Request example

<CodeGroup>

```bash curl
curl --get "https://fastapi-u8bm.onrender.com/api/tiktok/info" \
  --data-urlencode "url=https://www.tiktok.com/@user/video/7123456789012345678"
```

```javascript JavaScript
const params = new URLSearchParams({
  url: 'https://www.tiktok.com/@user/video/7123456789012345678'
});

const response = await fetch(
  `https://fastapi-u8bm.onrender.com/api/tiktok/info?${params.toString()}`
);

const { data } = await response.json();

// Choose quality: hdplay > play > wmplay
const videoUrl = data.hdplay || data.play || data.wmplay;
```

</CodeGroup>

## Response fields

<ResponseField name="success" type="boolean" required>
  `true` when TikWM returned valid data. On failure returns HTTP 400 with a `detail` field.
</ResponseField>

<ResponseField name="data" type="object" required>
  <Expandable title="properties" defaultOpen>
    <ResponseField name="id" type="string">
      Unique TikTok video ID.
    </ResponseField>
    <ResponseField name="title" type="string">
      Video caption text.
    </ResponseField>
    <ResponseField name="cover" type="string">
      Absolute URL of the video thumbnail image.
    </ResponseField>
    <ResponseField name="play" type="string">
      CDN URL for the standard-definition, watermark-free video. Always an absolute `https://` URL.
    </ResponseField>
    <ResponseField name="wmplay" type="string">
      CDN URL for the watermarked version of the video (includes the TikTok logo overlay).
    </ResponseField>
    <ResponseField name="hdplay" type="string">
      CDN URL for the HD version of the video (highest quality). Use this URL when available; fall back to `play` if empty.
    </ResponseField>
    <ResponseField name="music" type="string">
      CDN URL for the audio track extracted from the video as an MP3 file.
    </ResponseField>
    <ResponseField name="music_info" type="object">
      <Expandable title="properties">
        <ResponseField name="id" type="string">
          Unique identifier of the music track.
        </ResponseField>
        <ResponseField name="title" type="string">
          Music track title.
        </ResponseField>
        <ResponseField name="author" type="string">
          Artist or account that created the sound.
        </ResponseField>
        <ResponseField name="album" type="string">
          Album name. Empty string when not applicable.
        </ResponseField>
        <ResponseField name="cover" type="string">
          URL of the music cover art image.
        </ResponseField>
        <ResponseField name="duration" type="number">
          Duration of the music track in seconds.
        </ResponseField>
        <ResponseField name="original" type="boolean">
          `true` if the audio is an original sound created by the video author.
        </ResponseField>
        <ResponseField name="play" type="string">
          CDN URL to stream or download the music track as an MP3.
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="author" type="object">
      <Expandable title="properties">
        <ResponseField name="id" type="string">
          Unique TikTok user ID of the uploader.
        </ResponseField>
        <ResponseField name="unique_id" type="string">
          The TikTok username (handle) without the `@` prefix.
        </ResponseField>
        <ResponseField name="nickname" type="string">
          Display name of the author as shown on their profile.
        </ResponseField>
        <ResponseField name="avatar" type="string">
          URL of the author's profile picture.
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="duration" type="number">
      Video duration in seconds.
    </ResponseField>
    <ResponseField name="size" type="number">
      File size of the standard-definition video in bytes.
    </ResponseField>
    <ResponseField name="hd_size" type="number">
      File size of the HD video (`hdplay`) in bytes.
    </ResponseField>
    <ResponseField name="digg_count" type="number">
      Total number of likes (diggs) on the video at the time of the request.
    </ResponseField>
    <ResponseField name="comment_count" type="number">
      Total number of comments on the video.
    </ResponseField>
    <ResponseField name="share_count" type="number">
      Total number of times the video has been shared.
    </ResponseField>
    <ResponseField name="create_time" type="number">
      Unix timestamp (seconds since epoch) when the video was published.
    </ResponseField>
  </Expandable>
</ResponseField>

<Note>
  CDN URLs in `play`, `wmplay`, `hdplay`, and `music` all begin with `https://www.tikwm.com` and require a `Referer: https://www.tikwm.com/` header to fetch successfully. Use [`GET /api/stream`](/api-reference/stream) or [`POST /api/proxy-stream`](/api-reference/proxy-stream) to proxy these URLs rather than fetching them directly from a browser or React Native app.
</Note>
