The Social Media Downloader API lets you fetch video metadata and stream video files from TikTok, Instagram, YouTube, and over 1000 other platforms — all through a single HTTP API. You send a URL, and the API resolves the video, handles CDN authentication on your behalf, and streams the file directly to your client. No API key is required, and CORS is open so you can call it from any browser, mobile app, or server.

## Download architecture

Every download follows a two-step flow:

1. **Resolve** — Call `POST /api/get-direct-url` with a video URL. The API uses yt-dlp or the TikWM API to extract the real CDN URL and returns either a direct CDN link or a pre-built `/api/stream` proxy URL.
2. **Stream** — Hit `GET /api/stream` with the URL returned in step 1. The server fetches video bytes from the CDN and proxies them directly to your client. No file is stored on the server beyond the duration of the request.

For platforms like TikTok whose CDNs block direct client access, `force_backend_stream` is set to `true` in the response and the `direct_url` is already pointed at `/api/stream` — so your client just follows the URL without any extra logic.
